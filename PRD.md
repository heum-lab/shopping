# PRD.md — Product Requirements Document

## 제품명
**ONEPICKACOUNT CRM Admin**

## 버전
v1.0 (2026-04)

## 작성 목적
마케팅 대행사와 셀러 간 네이버 쇼핑 등 다채널 키워드 순위 작업을 관리·추적하는 내부 CRM 어드민 시스템의 요구사항을 정의한다.

---

## 1. 배경 및 목적

네이버 쇼핑, 플레이스, 블로그, 오늘의집, 카카오맵 등 다양한 플랫폼에서 광고 상품의 키워드 순위를 상승시키는 마케팅 서비스를 제공한다. 작업 등록부터 순위 추적, 기간 연장, 환불 처리까지 전체 워크플로우를 어드민에서 통합 관리해야 한다.

---

## 4. 핵심 기능 요구사항

### 4.1 네이버 쇼핑 상품 관리 (메인 기능)

#### 4.1.1 상품 등록
- 단건 등록: 폼을 통해 상품 정보 직접 입력
- 대량 등록: 엑셀 파일 업로드로 일괄 등록
- 필수 입력 항목: 대행사, 셀러, 키워드, 상품 MID, 상품 URL, 시작일, 종료일, 상태, 광고상품 타입, 유입수
- 선택 입력 항목: 서브 키워드1·2, 가격비교 MID·URL, 비고

#### 4.1.2 광고상품 타입
- 통검 (통합 검색)
- 쇼검 (쇼핑 검색)
- 통+쇼검
- 랜딩페이지
- 플러스스토어
- 기타유입 / 기타유입L
- 원픽플러스
- 팝콘
- 팝핀

#### 4.1.3 작업 상태 관리
| 상태 | 설명 |
|------|------|
| 대기 | 등록 완료, 작업 시작 전 |
| 작업중 | 유입 작업 진행 중 |
| 중지 | 작업 일시 중지 |
| 환불요청 | 셀러가 환불 요청 |
| 환불완료 | 환불 처리 완료 |
| 연장처리 | 기간 연장됨 |
| 작업완료 | 작업 정상 완료 |
| 삭제요청 | 삭제 요청됨 |

#### 4.1.4 일괄 처리 기능
- 선택 항목에 대한 일괄 상태 변경
  - 10일 연장처리
  - 7일 연장처리
  - 작업중 전환
  - 환불요청 전환
  - 삭제요청 전환
- 순위 일괄 조회

#### 4.1.5 순위 추적
- 최초순위 / 현재순위 / 어제순위 표시
- 순위 변동 표시: 상승(▲, 빨간색), 하락(▼, 파란색)
- 순위 상세 이력 팝업 조회 (`rank_popup`)

### 4.2 검색 및 필터링

| 필터 | 옵션 |
|------|------|
| 대행사 | 전체 또는 특정 대행사 선택 |
| 셀러 | 전체 또는 특정 셀러 선택 |
| 상태 | 전체 / 대기 / 작업중 / 중지 / 환불요청 / 환불완료 / 연장처리 / 작업완료 / 삭제요청 |
| 날짜 타입 | 시작일 / 종료일 / 주문일 / 환불요청일 |
| 간편 조회 | 오늘 / 어제 / 오늘·어제 / 7일전 / 1개월 / 3개월 / 6개월 / 12개월 / 이번달 / 이전달 |
| 검색어 | 셀러명 / 키워드 / 상품 MID |

### 4.3 엑셀 기능
- 엑셀 양식 다운로드
- 엑셀 파일 업로드로 대량 등록
- 검색 결과 엑셀 다운로드

### 4.4 업체 관리
- 대행사 관리: 대행사 등록·수정·삭제
- 셀러 관리: 셀러 등록·수정·삭제, 대행사 매핑

### 4.5 관리자 계정
- 로그인 / 로그아웃
- 비밀번호 변경
- 계정 정보 수정

---

## 5. 비기능 요구사항

- **페이지 로딩**: 목록 페이지 2초 이내 로딩
- **데이터 정합성**: 상태 변경 시 변경 이력(수정일) 기록
- **엑셀 처리**: 대량 등록 시 처리 중 로딩 표시 및 버튼 비활성화
- **반응형**: 가로 스크롤로 넓은 테이블 지원 (모바일 대응 최소화)
- **보안**: 관리자 페이지 로그인 필수, 세션 기반 인증

---

## 6. 목록 표시 설정

- 기본 표시 건수: 50개
- 선택 가능: 50 / 70 / 100 / 130 / 150개
- 정렬: 시작일, 종료일(환불요청일), 입금일 오름차순·내림차순 정렬 지원

---

## 7. 향후 개선 사항 (Backlog)

- 셀러 포털 (셀러 전용 뷰) 분리
- 실시간 순위 자동 수집 및 알림
- 채널별 통합 대시보드 (매출·작업 현황)
- 환불 처리 자동화 연동
